# Context Refresh Prompt

Use this when the governance rules have drifted out of context mid-project.
Paste the entire block. Do NOT use this as a reset — it recovers state first.

---

**Context Recovery — Do Not Reset**

Before doing anything else:

1. Read `memory.md` and report:
   - Current programme (if any) and current project name
   - Active stage
   - Stage approver
   - Last completed action
   - Next pending step or decision

2. Silently re-read `governance/00` through `governance/07` in order.

3. Print a compliance header:
   - Mode (Greenfield / Brownfield)
   - Active stage
   - Stage approver
   - Approval status of current stage
   - Allowed action scope for this turn

4. Ask: "Is there anything you need to clarify or correct before I continue?"

**Do not:**
- Re-ask questions already answered
- Re-run stages already approved
- Reset task numbering or restart memory

Resume exactly where we left off.
